2.1.1 Hardware-Voraussetzungen

Zur Installation des JDK ist ein vernünftig ausgestatteter PC mit Windows oder Linux oder ein zeitgemäßer Apple-Computer mit Mac OS/X erforderlich. Alternativ kann auch eine der vielen anderen Plattformen verwendet werden, auf die das JDK portiert wurde, beispielsweise eine der vielen UNIX-Varianten oder sogar Z/OS oder OS/2. Die meisten Beispiele in diesem Buch wurden auf unterschiedlichen PC-Systemen unter Windows 95, 98, NT oder XP entwickelt; später auch unter Vista, Mac OS/X und mit verschiedenen Linux-Derivaten.

Jeder aktuelle PC oder Mac erfüllt die Systemanforderungen, um Java-Programme zu entwickeln oder auszuführen. Während in früheren Versionen sowohl die Entwicklungswerkzeuge etwas träge waren als auch das Starten von Java-Programmen immer etwas dauerte, spielen diese Probleme bei den aktuellen Java-Versionen auf heutigen Systemplattformen keine Rolle mehr. Sollen integrierte Entwicklungssysteme anderer Hersteller verwendet werden, liegen die Hardware-Anforderungen meist deutlich oberhalb der Konfiguration beim Einsatz der Kommandozeilen-Werkzeuge. Aber die zum Zeitpunkt dieser Auflage aktuelle PC-Generation mit 2- bis 3-GHz-Mehrkern-Prozessoren und mehreren Gigabyte Hauptspeicher ist in jedem Fall ausreichend für praktisch alle Java-Entwicklungsarbeiten.

Die Installation der Java 6 Standard Edition erfordert inklusive Dokumentation etwa 350 MB Plattenspeicher, die der Version 7 unter Linux rund 600 MB. Insbesondere nach der (optionalen) Installation der Quelltexte und vor allem der Dokumentation sind sehr viele kleine Dateien vorhanden.

2.1.2 Installation

Wir wollen hier nur die Installation unter Windows beschreiben, die mit Hilfe eines Installationsprogramms menügesteuert erfolgt. Anschließend muss die Dokumentation des JDK installiert werden. Dies geschieht per Hand und erfordert ein Programm zum Entpacken der HTML-Dateien.

Installation des JDK

Die Java-Edition 7 befindet sich auf der DVD im Unterverzeichnis \install\java7. Falls die DVD nicht vorhanden ist, kann das JDK z.B. von http://jdk7.java.net geladen werden (ca. 80 MB Download zzgl. Dokumentation). Die Installation unter Windows ist sehr einfach und erfolgt in folgenden Schritten:

  • Zunächst muss in das Verzeichnis \install\java7 der DVD gewechselt werden.
  • Nun wird das Installationsprogramm jdk-7-windows-i586.exe (bzw. auf 64-Bit-Installationen jdk-7-windows-x64.exe) gestartet.
  • Nach Aufruf kann die Auswahl der zu installierenden Komponenten ausgewählt und das Installationsverzeichnis geändert werden. Wann immer nötig, gehen wir in diesem Buch davon aus, dass das JDK in das Verzeichnis \jdk1.7 installiert wurde. Die Vorgabe des Installers ist C:\Programme\Java\jdk1.7.0\, aber mit dem Button »Change« kann leicht ein anderes Verzeichnis ausgewählt werden. Die vorgegebene Zusammenstellung der Komponenten wollen wir nicht verändern. Nach Auswahl von »Next« wird die Installation begonnen.
  • Der anschließende Dialog dient zur Konfiguration der JRE-Installation, also der Java-Laufzeitumgebung. Auch hier sollte die Komponentenauswahl beibehalten werden. Und auch das Installationsverzeichnis C:\Programme\Java\jre7 lassen wir unverändert.
  • Nach einem weiteren »Next« werden die ausgewählten Komponenten des JRE installiert und das Installationsprogramm kann mit »Finish« beendet werden.

Wie erwähnt, wird mit dem JDK auch die Laufzeitumgebung JRE (Java Runtime Environment) installiert. Sie besitzt einen eigenen Deinstallationseintrag und befindet sich im Verzeichnis C:\Programme\Java\jre1.7.0. Für Arbeitsplätze, die nur das JRE benötigen (auf denen Java-Programme lediglich ausgeführt, nicht aber entwickelt werden sollen), kann das JRE auch alleine installiert werden. Es befindet sich ebenfalls auf der DVD zum Buch.
 Hinweis 

Alle Dateien befinden sich nun im ausgewählten Installationsverzeichnis und die Installation des JDK ist abgeschlossen. Um mit dem JDK arbeiten zu können, muss noch das Verzeichnis \jdk1.7\bin in den Suchpfad für ausführbare Dateien eingetragen werden.

Unter Windows ist der entsprechende Eintrag in den Umgebungsparametern der Systemkonfiguration vorzunehmen, unter XP beispielsweise unter »Einstellungen.Systemsteuerung.System« und dem Button »Umgebungsvariablen«. Informationen zu anderen Betriebssystemen und weitere Hinweise finden sich in den »Installation Notes« des JDK.

Installation der Dokumentation

Die Dokumentation des JDK besteht aus einer Sammlung von HTML-Dateien, die auf der DVD im Verzeichnis \install\java7 zu finden sind. Um sie zu installieren, muss die Datei jdk-7-fcs-bin-b147-apidocs-27_jun_2011.zip ausgepackt werden.

Die Dokumentation wurde bereits in das Unterverzeichnis docs verpackt. Falls das Installationsverzeichnis nicht geändert wurde, können die Dateien also ohne weitere Änderungen in das Verzeichnis c:\jdk1.7 ausgepackt werden. Alle erforderlichen Unterverzeichnisse werden dann automatisch angelegt. Zur Anzeige der Dokumentation ist ein Browser wie Chrome, Safari, Firefox oder Microsoft Internet Explorer erforderlich. Um leichter auf die Dokumentation zugreifen zu können, ist es sinnvoll, innerhalb des Browsers einen Verweis (Lesezeichen) auf das Hauptdokument \jdk1.7\docs\index.html anzulegen. Diese Datei kann zur Navigation auf alle Teile der Dokumentation verwendet werden und enthält zusätzlich eine Reihe von Verweisen auf externe Dokumente. Alternativ kann ein Icon auf dem Desktop angelegt werden, das den Browser mit dem Namen dieser Datei als Argument aufruft (die korrekte Schreibweise ist file:///C|/jdk1.7/docs/index.html). Auf dieselbe Weise kann ein Verweis auf die Datei \jdk1.7\docs\api\index.html angelegt werden, mit dem direkt in die API-Dokumentation gesprungen werden kann.

Installation der Quelltexte

Die J2SE 7.0 wird mit den vollständigen Java-Quelltexten ausgeliefert. Nicht enthalten sind dagegen die Quelltexte der Native Methods und der Entwicklungswerkzeuge. Nach der Installation des JDK liegen die Quelltexte im Archiv src.zip im Installationsverzeichnis. Sie sind zur Arbeit mit dem JDK nicht unbedingt nötig, können aber hilfreich sein, um weitergehende Fragen zu beantworten. Die Datei src.zip kann beispielsweise mit WinZip geöffnet und ihr Inhalt kann nach \jdk1.7\src ausgepackt werden. Alternativ kann natürlich auch das JDK-Werkzeug jar zur Extraktion verwendet werden. Seine Bedienung wird in Abschnitt 53.6 erklärt.

Deinstallation

Die Deinstallation des JDK ist denkbar einfach und kann über die Systemsteuerung erledigt werden. Bis auf die separat installierte Dokumentation und einige kleinere Dateien entfernt die Deinstallationsroutine die Bestandteile des JDK vollständig vom Rechner. Über einen weiteren Eintrag kann auch das JRE deinstalliert werden.
