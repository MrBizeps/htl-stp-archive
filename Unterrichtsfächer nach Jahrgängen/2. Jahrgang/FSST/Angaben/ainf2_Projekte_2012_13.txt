
AINF 2 -  Projektaufgaben 2012/13 : 

Die Aufgaben sind von jedem einzeln auszuarbeiten und 
als Dateiordner sowie als gedruckte Dokumentation abzugeben. 
Die gedruckte Dokumentation wird über die WORD-Vorlage 
"Projekt_Blatt.odt" erstellt und enthält -  
	die Aufgabenstellung, die Programmbeschreibung,
	das Programmlisting und das Benutzerhandbuch     
Die Ausführung sollte über Unterprogramme strukturiert erfolgen. 
	
Projektaufgaben - 
( Themenbereiche sind Unterprogramme, Grafikausgabe, Strings, Strukturen u. Dateiverabeitung ) 
   
1) Messdatenanalyse : 
   Messdaten sind in einer Textdatei zeilenweise mit Angabe 
   der laufenden Nummer, dem Messwert und Angabe der Einheit gespeichert. 
   Am Anfang der Datei befindet sich ein Kopfteil (Header) mit allg. Angaben. 
   (wie Titel und Datum der Messreihe, Kommentar zur Messung)  
   Der Datenteil wird dann über den Eintrag "Messwerte: " eingeleitet. 
   Die Messdaten sollen eingelesen, analysiert und grafisch dargestellt werden. 
   Die Analyse soll die Anzahl der Messdaten, den Mittelwert,
   den Maximal- und Minimalwert, sowie den RMS-Wert ermitteln und 
   als Protokolldatei im Textformat und als Webseite ausgeben. 
 
2) Speicher Oszilloskop 
   Messdaten werden von einer Binärdatei eingelesen und in ein dynamisches Array gespeichert.
   Die Signaldaten können dann ab einem Startzeitpunkt mit wählbarer Zeitdauer grafisch 
   in Form einer Digitalspeicher Oszilloskop Darstellung angezeigt und analysiert werden.   
  
3) Wörterbuch : 
   In zwei getrennten Dateien für Deutsch-Englisch u. Englisch-Deutsch 
   sind Vokabel folgend eingetragen : Vokabel Übersetzung   
   
   Über die Menüauswahl "Eintrag" kann ein neuer Eintrag ins Wörterbuch oder 
   das Bearbeiten der vorhandenen Einträge durchgeführt werden. 
   Über den Menüpunkt   "Deutsch->Englisch" kann das "Deutsch auf Englisch" Wörterbuch und 
   über den Menüpunkt   "English->German"   kann das "Englisch auf Deutsch" Wörterbuch abgefragt werden. 

4) Telefonbuch : 
   Name, Adresse, Telefonnummer(n) u. Handynummer 
   werden über eine Binärdatei verarbeitet. 
   Über eine Menüauswahl können neue Einträge, Änderungen  
   sowie die Benutzung (Abfrage u. Suchen) ausgeführt werden. 

5) Klassenkassa :   
   Eine Ein- u. Ausgaben Buchhaltung soll über eine Datenbank (über eine Binärdatei) abgewickelt werden. 
   Ein Datensatz enthält dazu folgende Datenfelder (Struktur) : 
   	  lfd.Nummer  Datum  Bezeichnung  Name  Betrag 
   Über eine Menüauswahl können neue Einträge hinzugefügt, 
   die Bearbeitung und die Abfrage von Einträgen durchgeführt, 
   sowie die Abfrage des Saldos über einen bestimmten Zeitraum ausgeführt werden.  
 
6) Vereinsdatenbank : 
   Die Daten von Vereinsmitgliedern werden über eine Binärdatei verarbeitet. 
   Die Datei enthält Name, Adresse, Geburtsdatum u. Telefonnummer, sowie auch 
   die Funktion im Verein (wie Obmann, Kassier, einfaches Mitglied)  
   und das Eintrittsdatum der Mitglieder. 
   Eintrag, Änderungen, Löschen und Abfragen von Einträgen erfolgt über 
   eine Menüauswahl. 

7) Quiz :  
   Quizfragen mit den richtigen Antworten werden in einer Binärdatei gespeichert.
   Die Einträge haben folgendes Format : Nummer Frage Antwort Bewertung  
   
   Über das Programm "quizadmin" können Quizfragen erstellt und bearbeitet werden.  
   Über das Programm "quiz" kann das Quizspiel gestartet werden. 
   Die Fragen mit der gewünschten Anzahl werden dann zufällig aus der Quizdatei gezogen, 
   die Antworten überprüft und  evtl. auch die Antwortzeiten gestoppt. 
   
8) Bibliothek :  
   Eine Bibliotheksverwaltung ist über ein Programm mit Binärdatei auszuführen. 
   Die Datensätze (Struktur) haben folgende Datenfelder : 
   	   Buchtitel Autor Kurzbeschreibung  Platznummer 
   	   	
   Über eine Menüauswahl kann die Bearbeitung der Datei und die Suche  
   nach Büchern über Titel oder Autor erfolgen. 
   
9) Wettbewerb : 
   Die Aufnahme der Teilnehmerdaten zur Auswertung von Wettbewerben wird 
   über eine Binärdatei durchgeführt. 
   Über eine Menüauswahl können zuerst die Teilnehmer erfasst, dann die 
   Ergebnisse eingetragen und zuletzt die Rangliste ausgegeben werden.   
   
10) Bauteil Kartei : 
   In einer Binärdatei werden die Artikeldaten und Lagermenge zu elektronischen Bauteilen gespeichert.  
   Über eine Menüauswahl kann die Erfasssung, Änderung und Abfrage von Bauteilen durchgeführt werden. 
   
11) elektronisches Klassenbuch :  
   Die Daten des Klassenbuchs sollen in einer Textdatei gespeichert werden. 
   Die Einträge werden folgend geführt :
   	  Datum Stunde Gegenstand Lehrer Anmerkungen fehlend: name1,name2,...name 
   Über eine Menüauswahl können Klassenbucheinträge, 
   das Blättern im Klassenbuch (Ausgabe einzelner Zeilen), 
   das Suchen nach den Einträgen an einem bestimmten Datum
   und die Abbrechnung der Fehlstunden (mit Ausgabe in eine Textdatei)
   ausgeführt werden. 
   
12) "login" Datenbank :  
   Die Daten für einen Benutzerzugang werden in einer Binärdatei gespeichert. 
   Ein Datensatz enthält dazu folgende Datenfelder : 
   	Login-Name MD5-Passwort  Beschreibung(vollständiger Name)  Gruppenzugehörigkeit  aktiviert(mit 0/1)   
   Das Administratorprogramm "useradmin" kann diese Daten eingeben und bearbeiten.   
   Das Anwendungsprogramm "userlogin" soll die Zugangsprüfung ausführen. 
   
13) Kontoführung :  
   Das Konto für Ein- und Ausgaben eines Betriebes wird über eine Binärdatei gespeichert. 
   Ein Datensatz enthält dazu folgende Datenfelder : 
   	  Datum   Beschreibung   Kontostelle   Betrag  
   Über eine Menüauswahl können Einträge hinzugefügt, geändert und 
   die Abfrage des Kontostandes durchgeführt werden.  
   
14) elektronische Beurteilungskatalog :  
   Die Daten eines Notenkatalogs sollen über Binärdateien verarbeitet werden. 
   Jede Klasse und jeder Gegenstand wird dazu in einer eigenen Datei gespeichert.
   Die Dateinamen werden nach folgendem Format "Noten_Klasse_Gegenstand.dat" erstellt.
   Ein Datensatz enthält die Datenfelder : 
    	Datum Stunde Lehrer Test_Note  MA_Note  Muendliche_Note   
   Über eine Menüauswahl können Noteneinträge u. Änderungen  
   und die Ermittlung des Gesamtnotenstand berechnet, angezeigt
   und als Textdatei ausgegeben werden. 
      
15) "Bilder" Archiv : 
   Ein Verzeichnis für Bilddateien soll über eine Binärdatei verarbeitet werden.
   Ein Datensatz enthält dazu folgende Datenfelder : 
   	  Bildtitel  Beschreibung  Verzeichnis  Dateiname   
   Das Erstellen, Ändern und Löschen von Einträgen soll über eine Menüauswahl erfolgen.  
   Die Bilddateien selbst befinden sich in Unterverzeichnissen zu einem gemeinsamen Hauptverzeichnis,
   das in der Konfigurationsdatei "BildarchivKonfig.txt" angegeben ist und bei Programmstart gelesen wird. 
  
16) Wave File Reader :   
  Wave-Dateien sollen geöffnet und gelesen werden. 
  Einige wichtige Daten vom Kopfteil (Head) sollen ausgegeben werden. 
  Die Samples (Signalwerte) sollen dann eingelesen 
  und mit wählbarer Zeitdarstellung als Grafik dargestellt werden. 

17) Schwingkreis Analyse  
  Die Berechnung der Resonanzfrequenz fr, Güte Q und Frequenzgang 
  eines RLC-Schwingkreises soll ausgeführt werden.
  Die Ergebnisdaten mit Frequenzgang werden grafisch am Bildschirm dargestellt 
  und zusätzlich werden die Ergebnisdaten mit Frequenztabelle als Textdatei ausgegeben.
 
18) Spiel - Gem Miner
	Zu Spielbeginn wird eine Welt mit Mineralien erschaffen. 
	Mit einem "Tunnelbohrer" kann nach den Mineralien gegraben werden 
	und die gewonnenen Mineralien kann man dann verkaufen. 
        Die Wirkung des Tunnelbohrers wird über Parameter, wie Bohrgeschwindigkeit, 
	Abnutzung und Treibstoff bestimmt. 
	Die Startkonfiguration des Spiels wird von einer Konfigurationsdatei im Textformat übernommen,
        der aktuelle Spielzustand kann in einer Binärdatei gespeichert und wiederum eingelesen werden. 

19) HTML- Generator
   Die aktuellen Analysedaten einer Meinungsumfrage sollen von einer CSV-Datei übernommmen  
   und als Webdatei ausgegeben werden. 
   Das Ausgabeformat der Webseite wird über ein HTML-Template und einer CSS-Datei vorgegeben.   


20) MyHome Datenbank
   eine Entwicklung einer eigenen (relationalen) Datenbank 
   mit einer C-Programmierschnittstelle (Funktionsbibliothek) 
   und einer "MyHome-Konsolenverwaltung" ist auszuführen. 


21) Direct Digital Synthesizer (DDS)
   Ein digitaler Signalgenerator ist mit grafischer Benutzeroberfläche zu entwickeln. 
   Die grafische Benutzeroberfläche soll mit der Klassenbibliothek "QT" erstellt werden. 
   Die Generatorstartkonfiguration wird dabei über eine Textdatei vorgenommen.
